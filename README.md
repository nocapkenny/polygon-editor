# Polygon Editor

Небольшой веб-редактор полигонов на нативном JavaScript и `canvas`.

Проект позволяет генерировать случайные полигоны, выбирать их на сцене, перемещать без пересечений, менять цвет, удалять, откатывать изменения и сохранять состояние между перезагрузками страницы.

## Возможности

- Генерация случайных полигонов прямо на холсте
- Защита от наложения фигур при генерации и перетаскивании
- Выбор полигона кликом
- Перемещение фигур мышью
- Изменение цвета выбранного полигона
- Удаление выбранного полигона или очистка всей сцены
- История изменений с `undo/redo`
- Импорт и экспорт сцены в `JSON`
- Сохранение текущей сцены в `localStorage`

## Стек

- JavaScript ES Modules
- Web Components
- HTML5 Canvas
- Webpack
- CSS

## Запуск проекта

Установить зависимости:

```bash
npm install
```

Запустить локальный dev-сервер:

```bash
npm start
```

Собрать production-версию:

```bash
npm run build
```

Опубликовать содержимое `dist` через GitHub Pages:

```bash
npm run deploy
```

Если в PowerShell заблокированы `.ps1`-скрипты, вместо `npm` можно использовать `npm.cmd`.

## Как пользоваться

1. Нажмите кнопку добавления, чтобы сгенерировать новый полигон.
2. Кликните по фигуре, чтобы выбрать её.
3. Перетаскивайте выбранный полигон мышью по холсту.
4. Используйте color picker для изменения цвета выбранной фигуры.
5. Удаляйте выбранную фигуру или очищайте всю сцену через панель инструментов.
6. Импортируйте и экспортируйте сцену в JSON-файл.

## Горячие клавиши

- `Ctrl + Z` — отмена действия
- `Ctrl + Y` — повтор действия
- `Delete` — удалить выбранный полигон

## Сохранение состояния

Текущее состояние сцены сохраняется в `localStorage` браузера и восстанавливается при следующем открытии страницы.

Дополнительно сцену можно экспортировать в `JSON` и позже импортировать обратно.

## Структура проекта

```text
src/
  components/   UI-компоненты приложения
  core/         состояние, действия, история, рендер, сохранение
  styles/       глобальные и компонентные стили
  utils/        вспомогательные функции
```

## Архитектура

Проект построен вокруг простого глобального `state`-объекта и механизма подписок.

- `state.js` хранит текущее состояние приложения
- `actions.js` описывает изменения состояния
- `history.js` управляет `undo/redo`
- `renderer.js` отвечает за отрисовку фигур на `canvas`
- `save.js` работает с `localStorage` и JSON-импортом/экспортом

## Ограничения

- История `undo/redo` не восстанавливается после перезагрузки страницы
- Импорт ожидает корректный JSON со списком полигонов
